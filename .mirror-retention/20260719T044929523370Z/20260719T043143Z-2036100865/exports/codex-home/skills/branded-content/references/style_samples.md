# 空格的写作风格样本

> 以下是空格的代表性文章，创作商单内容时必须严格对齐这些文章的风格。
> 重点关注：句式节奏、段落长度、叙事结构、产品分析方式、开头结尾手法。

## 样本一：飞书 aily 产品分析（产品体验型）

飞书发布原生"龙虾" aily，这才看懂了飞书的 AI 布局

今天看了飞书 2026 新品发布会。

核心产品是 aily，飞书原生的 AI 助手，就是飞书官方出的 OpenClaw。

估计国内装 OpenClaw 的人，大部分都把对话接到了飞书里。

这和飞书一直以来的开放定位有很大关系，开放 API、开放机器人、开放文档能力。

这让我的龙虾接了飞书官方的产品，实现一句话给龙虾，就推送本地文档到飞书再来协同

我还借助飞书的开放能力给产品做数据后台支持，飞书一直是我作为个人开发者的基础设施。

所以在飞书内做 Agent，不意外且我对此充满期待。

整个发布会过程也是惊喜满满

三个点让我觉得值得关注：

**1，激活快到离谱。**

打开飞书 → 搜索"aily" → 点击激活。10 秒，它出现在你的通讯录里。起个名字，设定身份，完事。

不用配环境、不用写代码、不用看教程。谢欣现场演示那段，从头到尾不到一分钟。OpenClaw 装环境装到一半放弃的人，看到这里估计会沉默。

**2，打通了飞书生态。**

aily 是飞书原生的，它能直接访问你的文档、日历、群聊、项目、多维表格。

走的是你自己的权限体系，发布会上演示了几个场景：把访谈记录扔给 aily，说"帮我分类整理"，它自动建多维表格、自动填内容、自动摘要。

给它设个定时任务，每天晚上去群里扫用户反馈，按优先级分好类，第二天早上报给你。

这种"直接在企业上下文里干活"的能力，OpenClaw 要配一堆 MCP 才能勉强做到，aily 开箱就有。

**3，官方推出了 skill 技能库**

飞书结合自己在企业方面的知识，做了官方推出的技能库，直接发给 aily，就能让他安装官方的技能库，这对于企业、个人都省去了自己创作技能的麻烦，而且可以保证企业的数据安全。

**4，有记忆，越用越懂你。**

谢欣说他养了两个月，现在 aily 的输出已经非常贴合他的工作方式了。工作习惯、沟通偏好、你惯用的格式，用久了它都记。这不是工具，是搭档。搭档需要磨合期，但磨合完了效率是指数级的。

现在我飞书里又多了一只新的虾，速度快、体验好。

1. 打开飞书，搜索"aily"
2. 点击激活，起名字，设身份
3. 开始对话，让它帮我操作多维表格或文档

最近有很多应用把 GUI 改成 CLI，为 Agent 重新设计了一套企业平台。

今天飞书发了 aily，使用了另一种方式，用原生集成 + 零门槛的方式，把 Agent 直接塞进了 2 亿用户的通讯录里。

飞书在做"每个人都能开箱即用的 AI 搭档"，重的是用户体验，让用户指挥龙虾来操作生态内的工具

企业软件正在从"为人设计"变成"为人和 Agent 同时设计"。

让 Agent 在企业场景里真正跑起来。

飞书今天亮了一手好牌。接下来看谁先把牌打好。

## 样本二：钉钉悟空 行业分析（行业分析型）

昨天看了阿里 AI 钉钉 2.0 年度发布会。

总结下来就两件事：

1，成立 Alibaba Token Hub 事业群（ATH），CEO 吴泳铭亲自挂帅，通义实验室（创造Token）、MaaS业务线（输送Token）以及千问、悟空等事业部（应用Token）放置在同一个体系内。

2，发布悟空，把钉钉从底层代码推倒重写，从 GUI 界面改成 CLI 命令行。悟空就是为 Agent 重新设计的钉钉，直接内置到超2000万企业组织的钉钉中。

两者结合起来，阿里是在用ATH这条"公路"把从造Token到用Token的整条链串起来了。这样一来，悟空想要调动集团资源，就像在家里开灯用水一样自然。电商、阿里云这些核心能力，统统给悟空开了"超级VIP通道"。

这件事放在创业公司身上很轻松，openai、anthropic 已经是这样在做了。

对阿里来说，要实现模型研发到产品应用，再到组织内外的商业化。

这是必经之路，也是我们一直关注的一个问题

企业该如何结合 AI 发展？

Anthropic 的模式值得参考：

Anthropic 从第一天就死磕企业客户这条路。Claude在企业场景里越磨越锋利，企业用得越爽，付费越大方，钱又砸回研发反哺模型。这个"能力越强→企业越爱→赚得越多→做得更强"的飞轮，在B端天然就跑通了。到2025年，格局变了，硅谷最炙手可热的AI公司，换Anthropic来坐了。

悟空在做的事也是每一个做企业软件值得关注的点，过去十年，SaaS 工具是为「人」设计的——界面、按钮、操作流，都假设使用者是人类。

但现在，Agent 才是真正的使用者。

如果 Agent 还要模拟人类点击才能操作企业系统，效率会很低，也容易出错。

悟空就是在做这件事：把钉钉重新设计成一个「Agent 原生」的企业平台。

Agent 直接调用权限、直接执行流程、直接核算成本。

钉钉 11 年积累的组织架构、审批链、权限体系，悟空可以通过 CLI 实现。

阿里今天亮出了完整的牌面。

接下来看执行。

## 样本三：秒哒短剧应用（方法论总结型）

最近在探索短剧生成相关的方向，刚好想怎么把自己积攒的经验做成一个产品。

只要集成文字、图片、视频就能让短剧制作这个门槛变得很低。

整体逻辑就是：一句话描述故事——分镜描述生成——分镜图片制作——分镜视频制作

它要把文字、图片、甚至视频都融到一起。所以我没有一上来就硬做，而是先拆开来，分别试了试文字生成和图片生成的效果怎么样。

我先做了两个小应用练手：一个叫「动物生命周期」，就是把文字转成 SVG 代码来做可视化，偏娱乐学习的；

另一个是「小红书封面图制作」，把文字直接转成图片，这个对我自己的日常工作是真有提效的。

这两个都是通过和龙虾对话做出来的，用的是秒哒的 Skill。做完之后都能跑通，说实话还是挺惊喜的——毕竟这是我第一次纯靠对话就做出了这样的产品。

跑通这两个之后，我对做短剧应用就更有信心了。然后也是先和龙虾对话，确定大致方向，再到秒哒平台上去详细设计，主要是配置图片和视频的插件。差不多 10 轮对话，短剧脚本制作这个应用就出来了，最后成功上线。

这个应用核心想解决的是什么呢？就是降低剧本和分镜创作的门槛。

你有个灵感，一次可以发散出几十个剧本和分镜，然后再从里面挑最合适的，效率比从零开始写高太多了。

最后我总结了一套做这类应用的通用方法论，不管你是做文字、图片还是视频，其实都是三步：

第一步，设计输入的容器。可以是一句话、一个表单、一个文件、一张图片，总之就是用户把东西丢进来的入口。

第二步，设计模型处理的方式。最简单的就是一段提示词加上用户的输入发给模型；复杂一点的，可以先给模型 A 处理，再交给模型 B 接力。

第三步，设计输出的容器。如果输出是 SVG，就用前端渲染做可视化；如果是 Markdown，就以文章形式呈现；如果是结构化数据，就用表格展示。

记住这三步——输入容器、模型处理、输出容器，基本上你就能把大多数 AI 应用的框架搭起来了。

## 风格提炼要点

从以上样本中可以提炼出空格写作的核心模式：

### 开头模式
- 事件驱动："今天看了XX发布会" / "看到XX出了个新产品"
- 直接切入，第一句话就告诉读者发生了什么
- 不要铺垫，不要"大家好今天来聊聊"

### 叙事推进
- 先给背景和个人关联（为什么我关注这个）
- 再拆解具体亮点（用数字编号：1，2，3...）
- 每个亮点用加粗小标题 + 具体描述
- 穿插个人体验和判断

### 分析视角
- 不是说明书式地列功能，而是"它做对了几件事"
- 会和同类产品/过去的方式做对比
- 会给出"为什么这件事重要"的判断

### 结尾模式
- 拉到行业高度给一句判断
- "XX亮了一手好牌，接下来看执行/看谁先把牌打好"
- 或总结一个方法论/框架

### 节奏特征
- 一句话一行，段落极短
- 短句 → 短句 → 稍长的句子收住，形成呼吸感
- 关键判断单独成段
