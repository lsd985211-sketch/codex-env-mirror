# 商单内容制作工作流

五个阶段：Brief 分析 → 角度策划 → 素材整合 → 风格对齐 → 内容输出。

## 文章类型判断

根据 brief 内容和用户素材，判断属于哪种类型：

| 类型 | 特征 | 文章骨架 |
|------|------|----------|
| **产品实测型** | 用户有测试截图/demo/对比数据 | 多个 case 横评，截图对比，自然植入 |
| **事件解读型** | 产品发布/行业事件为核心 | 事件 → 产品力 → 评价 → 趋势 |
| **场景体验型** | 强调使用场景和痛点解决 | 痛点 → 方案 → 体验 → 推荐 |

**产品实测型是最常见也最有效的商单形式。** 大多数情况默认推荐此类型。

## Stage 1: Brief 分析

**用户发送商单 brief 后，立刻分析提取：**

1. **产品/品牌**：什么产品，核心卖点
2. **传播目标**：品牌方想传达什么
3. **必含要素**：关键词、链接、话术
4. **禁忌红线**：不能提的竞品、不能用的说法
5. **发布平台**：公众号/小红书/视频脚本

提取完毕后，用简洁列表展示确认。

## Stage 2: 角度策划

**提供 3 种差异化方案，每种包含：**

- **角度名称**（一句话概括）
- **切入方式**（怎么开头、怎么引出产品）
- **内容结构**（3-5 个章节标题）
- **需要的素材**（需要用户提供什么测试内容）
- **传播亮点**（为什么能传播）

### 方案设计原则

1. **至少一个「实测横评」方案**：对比竞品，用 demo 说话——这是转化率最高的形式
2. **至少一个「热点借势」方案**：绑定当前行业事件或趋势
3. **方案之间要有明显区分**

### 产品实测型方案的标准结构（最常用）

参考成功案例提炼的骨架：

```
1. 开头：行业背景 / 个人状态 / 热点事件（1-2 段，快速进入）
2. Case 1：入门测试（让读者快速看到产品能力）
3. Case 2-4：核心测试场景（难度递进，有对比有截图）
4. Case 5+：杀手级场景（最能体现差异化优势的 case）
5. 总结：个人真实判断 + 产品信息自然植入
```

**关键：每个 case 的结构是「提示词 → A 模型结果 → B 模型结果 → 点评」，用截图/demo 说话。**

### 实测文章的核心技巧

- **坦诚弱点**：不要全是好话，承认不足反而更可信（「这块确实 Claude 还是更强」）
- **个人真实反应**：「说实话我被惊到了」「坐在电脑前愣了一会儿」比任何数据都有说服力
- **成本对比**：价格优势是很强的传播素材（「便宜 X 倍」「开源免费」）
- **渐进惊喜**：测试难度递进，最后一个 case 给读者最大惊喜
- **自然植入**：产品链接/套餐在测试过程中带出，不要放在文末硬推

## Stage 3: 素材整合

**用户选定方案后，根据方案类型索要素材：**

### 产品实测型需要的素材

- 测试了哪些场景/case？（提示词、截图、对比结果）
- 有没有对比竞品的结果？
- 有没有让你惊喜或失望的点？
- 跑测试的平台/工具是什么？

### 事件解读型需要的素材

- 事件相关的数据/截图/公告
- 海内外评价/社区反应
- 你个人的观点和判断

**收到素材后：**

1. 提取关键数据和亮点
2. 按方案结构分配素材到对应章节
3. 输出完善后的详细大纲（含每章使用的具体素材）
4. 展示给用户确认

**素材不全也能写，不卡在收集阶段。**

## Stage 4: 风格对齐

**询问用户是否有参考文章。**

如有，分析：
- 句式特点（短句/长句、口语/书面）
- 产品植入方式（测试过程中自然带出 vs 文末集中推荐）
- 对比表达（坦诚弱点程度、夸赞尺度）
- 个人化程度（第一人称体验 vs 客观报道）

告知「参考风格已学习」，进入下一阶段。

## Stage 5: 内容输出

**调用 doc-coauthoring skill 的 Stage 2 流程输出全文。**

## 核心原则

### 产品实测文章的金标准

- **读起来像博主真的在用**，不像品牌方写的
- Demo 和截图是骨架，文字是连接和点评
- 坦诚 > 吹捧，真实体验 > 参数罗列
- 每篇至少一个「金句」可被单独截图传播

### 效率优先

- 每阶段结束直接进入下一阶段
- 用户说「继续」就往下走
- 素材不全也能写

### 常见反模式（避免）

- ❌ 通篇堆砌参数和跑分数据
- ❌ 产品信息集中在文末硬推
- ❌ 只说优点不提缺点
- ❌ brief 中的官方话术原封不动搬入文章
- ❌ 「不是…而是…」「这意味着什么？」等 AI 味句式
